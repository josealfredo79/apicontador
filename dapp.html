📚 ¿Cómo funciona? (expandir)
Registro con Passkey: 1. Navegador → POST /auth/register/begin → Server 2. Server genera challenge (número random único) 3. Dispositivo pide biometría (huella / Face ID) 4. WebAuthn crea par de claves: pública + privada 5. Clave PRIVADA queda en el dispositivo (nunca sale) 6. Clave PÚBLICA + firma → POST /auth/register/complete 7. Server verifica y guarda la clave pública 8. Se emite token de sesión (24h) Login con Passkey: 1. Server genera nuevo challenge (anti-replay) 2. Dispositivo firma el challenge con clave privada 3. Server verifica con la clave pública guardada 4. Si válido → nuevo token de sesión Smart Wallet (concepto Soroban): Cada usuario tiene un walletId único que simula la dirección de un contrato Soroban. En producción se desplegaría passkey-kit de Stellar y las TXs serían firmadas directamente desde el contrato.
