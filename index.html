¿Qué ocurre al pulsar +?
  1. API verifica el Bearer Token (middleware requireAuth)
  2. Obtiene el número de secuencia de la cuenta Stellar
  3. Construye una transacción Soroban que llama a increment()
  4. Simula la TX para calcular el fee exacto
  5. Firma con la SECRET_KEY del servidor
  6. Envía a la red → confirmación en ~5 segundos
  7. Retorna el nuevo valor + txHash

¿Qué es una API REST?

Una API es como el mesero de un restaurante: tú pides algo, él va a la cocina y te lo trae. REST define las reglas de cómo hacer esos pedidos usando HTTP.

☁️ Deploy en Railway

Railway detecta automáticamente proyectos Node.js y los despliega sin configuración compleja.

La app ya está desplegada y funcionando
URL en producción: https://api-contador-production.up.railway.app

Instalar Railway CLI y hacer login

Terminal
npx @railway/cli login   # abre el navegador para autenticarse
2

Crear el proyecto y hacer el primer deploy

Terminal
npx @railway/cli init    # nombre del proyecto
npx @railway/cli up      # sube el código y construye

npx @railway/cli domain  # genera el dominio público

Configurar variables de entorno

Terminal
npx @railway/cli variables set \
  CONTRACT_ID=TU_CONTRACT_ID \
  PUBLIC_KEY=TU_PUBLIC_KEY \
  SECRET_KEY=TU_SECRET_KEY \
  RPC_URL=https://soroban-testnet.stellar.org \
  NETWORK_PASSPHRASE="Test SDF Network ; September 2015" \

RP_ID es crítico para las Passkeys
Debe ser exactamente el dominio sin protocolo. Si no coincide con la URL real de la app, WebAuthn rechaza la autenticación con error de origen.
4

Redeploy y ver logs

Terminal
npx @railway/cli up --detach   # sube sin bloquear la terminal
npx @railway/cli logs          # ver logs en tiempo real

Debugging

🔧 Errores Comunes y Soluciones

CONTRACT_ID no configurado
El archivo .env no existe o le falta CONTRACT_ID. Ejecuta: cp .env.example .env y edita con tus datos.
Account not found (Stellar)
La cuenta no está fondeada en Testnet. Ejecuta: stellar keys fund MI-CUENTA --network testnet
ECONNREFUSED (conexión rechazada)
El servidor no está corriendo. Ejecuta npm start en una terminal y déjala abierta.
Transaction failed (Soroban)
CONTRACT_ID incorrecto, cuenta sin XLM, o contrato no desplegado. Verifica el txHash en stellar.expert.
Passkeys no funcionan en Railway
El RP_ID debe ser el dominio exacto sin https://. Si no coincide, WebAuthn rechaza con error de origen.
Se perdieron los usuarios registrados
Los usuarios están en memoria RAM. Al reiniciar se pierden. Solución en producción: conectar PostgreSQL (disponible gratis en Railway).
