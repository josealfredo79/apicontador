📍 ¿Qué es un Endpoint?

🔑 Definición Simple

Un endpoint es como una puerta específica en un edificio (la API). Cada puerta hace algo diferente:

🏠 Analogía del Edificio

Imaginá un hospital con diferentes consultorios:
• /cardiología → Ver cardiologist
• /pediatría → Ver pediatrician
• /emergencias → Atención de urgencia

🔧 Endpoints de Nuestra API

Nuestra API tiene estas "puertas":

/contador → Leer el valor actual
/contador/increment → Sumar 1
/contador/decrement → Restar 1
/contador/reset → Reiniciar a 0
🔤 Partes de un Endpoint
GET http://localhost:3000/contador
  • GET = El método (qué hacer: leer, escribir, etc.)
  • http://localhost:3000 = La dirección del servidor
  • /contador = El endpoint específico (la "puerta")

📌 Resumen

Endpoint = URL específica que hace una acción particular en la API.

Es como una dirección exacta: /contador/increment es el lugar donde vas a "incrementar el contador".

💻 Cómo Consumir la API

🔑 Concepto Clave

Consumir una API significa pedirle datos o enviarle acciones desde tu código JavaScript, como si fuera una conversación:

// Tu código pregunta: fetch('/contador') // La API responde: { valor: 5, success: true }

1️⃣ Con fetch() (Nativo del navegador)

📋 Ejemplo con fetch() 
// 📥 LEER el valor del contador
async function obtenerContador() {
    const respuesta = await fetch('http://localhost:3000/contador');
    const datos = await respuesta.json();
    console.log('Valor actual:', datos.valor);
    return datos;
}

// ➕ INCREMENTAR el contador
async function incrementar() {
    const respuesta = await fetch('http://localhost:3000/contador/increment', {
        method: 'POST'
    });
    const datos = await respuesta.json();
    console.log('Nuevo valor:', datos.valor);
    return datos;
}

// ➖ DECREMENTAR el contador
async function decrementar() {
    const respuesta = await fetch('http://localhost:3000/contador/decrement', {
        method: 'POST'
    });
    const datos = await respuesta.json();
    console.log('Nuevo valor:', datos.valor);
    return datos;
}

// 🔄 RESETEAR a 0
async function resetear() {
    const respuesta = await fetch('http://localhost:3000/contador/reset', {
        method: 'POST'
    });
    const datos = await respuesta.json();
    console.log(datos.mensaje);
    return datos;
}

2️⃣ Con Axios (Librería popular)

📋 Ejemplo con Axios 
// Instalar: npm install axios

import axios from 'axios';

// 📥 Obtener valor
const datos = await axios.get('http://localhost:3000/contador');

// ➕ Incrementar
const resultado = await axios.post('http://localhost:3000/contador/increment');

// 👍 Más fácil, el resultado ya viene en datos.data
console.log(resultado.data.valor);

3️⃣ Con jQuery

📋 Ejemplo con jQuery 
// 📥 Obtener valor
$.get('http://localhost:3000/contador', function(data) {
    $('#valor').text(data.valor);
});

// ➕ Incrementar
$.post('http://localhost:3000/contador/increment', function(data) {
    alert(`Nuevo valor: ${data.valor}`);
});

📊 Cuándo usar cada método

Método Cuándo usarlo
fetch() Proyectos modernos, no necesita librería
Axios Proyectos grandes, quiere más funcionalidades
jQuery Proyectos legacy o aprendizaje básico

🔄 Ciclo Request-Response (Detallado)

🌐 Flujo Completo

1. Petición:      NAVEGADOR → API (/contador)
2. Proceso:       API → SOROBAN RPC → CONTRATO
3. Ejecución:     CONTRATO retorna valor
4. Respuesta:     CONTRATO → RPC → API → NAVEGADOR
1 Cliente → API

Tu navegador envía una petición HTTP a http://localhost:3000/contador

2 API → Blockchain

El servidor transforma la petición en una transacción Soroban y la envía a TestNet

3 Blockchain procesa

El smart contract ejecuta la lógica (get, increment, decrement, reset)

4 Respuesta ← Cliente

El resultado vuelve por el mismo camino hasta tu navegador

🏗️ Estructura de una API REST

📐 Partes de una API

1️⃣ Cliente

Tu navegador, app o código que hace las peticiones

fetch('/contador')
2️⃣ Servidor

Recibe peticiones y devuelve respuestas

Express.js
3️⃣ Rutas

Los endpoints (las "puertas" específicas)

/contador/increment
4️⃣ Métodos

Qué acción realizar (GET, POST, etc.)

GET, POST, PUT, DELETE

🔗 Anatomía de una URL de API

https://localhost:3000/contador/increment

│              │              │         │          │
│              │              │         │          └── 3. RECURSO (acción)
│              │              │         └────────────── 2. ENDPOINT (ruta)
│              │              └──────────────────────── 1. PUERTO
│              └─────────────────────────────────────── SERVIDOR
└────────────────────────────────────────────────────── PROTOCOLO

📋 Métodos HTTP

Método Descripción Ejemplo
GET Leer / Obtener datos Ver el contador
POST Crear / Enviar datos Incrementar (+1)
PUT Actualizar datos Cambiar valor
DELETE Borrar datos Eliminar contador

💬 Formato de Datos: JSON

// La API siempre devuelve JSON (como objetos de JS) { "valor": 5, "success": true, "txHash": "abc123..." }

🔌 Endpoints

Obtener Valor Actual

GET /contador

Retorna el valor actual del contador.

curl http://localhost:3000/contador
// Respuesta { "valor": 5, "success": true }

Incrementar

POST /contador/increment

Suma 1 al contador y retorna el nuevo valor.

curl -X POST http://localhost:3000/contador/increment
// Respuesta { "valor": 6, "txHash": "a1b2c3...", "success": true }

Decrementar

POST /contador/decrement

Resta 1 al contador (mínimo 0).

curl -X POST http://localhost:3000/contador/decrement

Resetear

POST /contador/reset

Reinicia el contador a 0.

curl -X POST http://localhost:3000/contador/reset

Estado del Servidor

GET /health

Verifica que el servidor esté corriendo.

curl http://localhost:3000/health

🔧 Solución de Problemas

Error: CONTRACT_ID not configured

Asegúrate de que el archivo .env tenga el CONTRACT_ID del contrato desplegado.

Error: Account not found

La cuenta no está fondeada en testnet. Usa:

stellar keys fund mi-cuenta --network testnet

Error: Connection refused

El servidor no está corriendo. Verifica con:

npm start

Error: Transaction failed

Puede ser falta de balance o el contrato no está desplegado. Verifica:

  • Que el contrato esté desplegado en testnet
  • Que haya suficiente XLM en la cuenta
  • Que el CONTRACT_ID sea correcto
