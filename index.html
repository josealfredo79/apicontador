⚙️ Instalación

4. Estructura del Proyecto

🔌 Endpoints

Obtener Valor Actual

GET /contador

Retorna el valor actual del contador.

curl http://localhost:3000/contador
// Respuesta { "valor": 5, "success": true }

Incrementar

POST /contador/increment

Suma 1 al contador y retorna el nuevo valor.

curl -X POST http://localhost:3000/contador/increment
// Respuesta { "valor": 6, "txHash": "a1b2c3...", "success": true }

Decrementar

POST /contador/decrement

Resta 1 al contador (mínimo 0).

curl -X POST http://localhost:3000/contador/decrement

Resetear

POST /contador/reset

Reinicia el contador a 0.

curl -X POST http://localhost:3000/contador/reset

Estado del Servidor

GET /health

Verifica que el servidor esté corriendo.

curl http://localhost:3000/health

  1. GET /contador (verificar valor inicial)
  2. POST /contador/increment (3 veces)
  3. GET /contador (debe ser 3)
  4. POST /contador/decrement (2 veces)
  5. GET /contador (debe ser 1)
  6. POST /contador/reset
  7. GET /contador (debe ser 0)

🔧 Solución de Problemas

Error: CONTRACT_ID not configured

Asegúrate de que el archivo .env tenga el CONTRACT_ID del contrato desplegado.

Error: Account not found

La cuenta no está fondeada en testnet. Usa:

stellar keys fund mi-cuenta --network testnet

Error: Connection refused

El servidor no está corriendo. Verifica con:

npm start

Error: Transaction failed

Puede ser falta de balance o el contrato no está desplegado. Verifica:

  • Que el contrato esté desplegado en testnet
  • Que haya suficiente XLM en la cuenta
  • Que el CONTRACT_ID sea correcto
